# SEO Audit

## 触发条件
1. 用户希望提升网站在搜索引擎（Google, Baidu, Bing）中的排名。
2. 网站流量出现异常下跌，需要排查技术或内容问题。
3. 新网站上线前，需要进行全面的搜索引擎友好性检查。
4. 现有内容无法被搜索引擎正常收录或索引。
5. 需要对比竞品 SEO 表现并制定超越策略。

## 核心能力

### 1. 技术 SEO 审计 (Technical SEO)
检查搜索引擎爬虫是否能顺利抓取并索引页面：
- **robots.txt**：验证是否阻止了不该阻止的页面（如登录页、后台）。
- **Sitemap.xml**：检查站点地图是否包含所有核心页面，且格式符合标准。
- **Canonical Tags**：防止因 URL 参数导致的内容重复，确保权重复合到单一 URL。
- **Hreflang Tags**：对于多语言网站，确保不同语言版本的正确关联。
- **SSL/HTTPS**：验证全站是否强制 HTTPS。

### 2. 页面内 SEO (On-page SEO)
针对具体页面的内容和结构进行优化：
- **标题 (Title Tags)**：长度建议在 50-60 字符，包含核心关键词。
- **元描述 (Meta Description)**：撰写吸引点击的摘要（150-160 字符）。
- **内容结构 (H1-H6)**：确保一个页面只有一个 H1，且层级逻辑清晰。
- **关键词密度与布局**：检查 LSI（潜在语义索引）关键词的自然分布。
- **图片 Alt 属性**：确保所有图片都有描述性文本。

### 3. 核心网页指标 (Core Web Vitals)
评估页面加载速度与用户体验：
- **LCP (Largest Contentful Paint)**：最大内容渲染时间（< 2.5s）。
- **FID (First Input Delay)**：首次输入延迟（< 100ms）。
- **CLS (Cumulative Layout Shift)**：累计布局偏移（< 0.1）。
- **移动端友好性 (Mobile-Friendly)**：响应式设计检查、元素间距及字体大小评估。

### 4. 结构化数据 (Schema.org)
- **JSON-LD 配置**：为文章、产品、FAQ、评价、面包屑导航生成结构化数据代码，提升搜索结果中的富媒体展示（Rich Snippets）。

### 5. 内外链分析
- **内链结构**：构建合理的“中心-辐射”结构（Hub and Spoke），确权重流向核心页面。
- **外链质量**：识别并建议清理低质量或恶意垃圾外链。
- **断链检查**：扫描并修复全站 404 错误。

### 6. 竞品 SEO 对标 (Benchmarking)
- **关键词重叠度分析**：识别竞争对手排名靠前但己方未覆盖的关键词（Content Gap）。
- **反向链接对标**：寻找高权重的行业外链机会。

## 边界与限制
1. **黑帽 SEO 禁令**：拒绝执行任何包含关键词堆砌、隐藏文本、链接农场、恶意跳转等违反搜索引擎指南的行为。
2. **实时数据延迟**：由于搜索引擎索引更新有滞后，SEO 优化的效果通常需要数周至数月才能显现。
3. **内容生成限制**：生成的 SEO 内容必须基于真实信息，不得编造事实以骗取点击（Clickbait）。
4. **工具依赖**：深度的权重分析（DA/PA）需结合外部专业工具（如 Ahrefs, Semrush, Google Search Console）的数据。

## 最佳实践准则
- **用户体验第一**：为人类写内容，为机器做优化。
- **原创性优先**：避免采集和低质量洗稿。
- **持续监测**：SEO 是一场马拉松，而非百米冲刺。
