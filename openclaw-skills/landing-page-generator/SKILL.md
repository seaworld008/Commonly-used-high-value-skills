# Landing Page Generator

## 触发条件
1. 用户需要为新产品、新服务或特定营销活动快速搭建一个高转化率的落地页（Landing Page）。
2. 用户已提供初步的产品设想，但缺乏页面结构设计和吸引人的营销文案。
3. 需要快速输出 React/Next.js + Tailwind CSS 的代码原型，以便立即部署测试。
4. 进行页面优化（CRO）时，需要参考行业最佳实践的布局和元素。
5. 需要生成多种版本的文案进行 A/B 测试。

## 核心能力

### 1. 落地页标准结构 (High-Conversion Structure)
包含以下核心模块（按典型浏览顺序排列）：
- **Hero Section (英雄区)**：强有力的主标题（H1）、清晰的副标题（Value Prop）和醒目的 CTA 按钮。
- **Pain Points & Solution (痛点与方案)**：描述用户当前面临的问题，并展示产品如何解决它（PAS 框架）。
- **Features & Benefits (功能与收益)**：不仅展示“有什么功能”，更强调“给用户带来什么好处”。
- **Social Proof (社交证明)**：客户评价、媒体报道徽标、用户数量统计。
- **Process/How it Works (运作方式)**：简单 3-4 步说明如何开始使用。
- **Pricing/Offer (价格与优惠)**：清晰的价格体系或限时优惠。
- **FAQ (常见问题)**：消除用户最后的疑虑。
- **Final CTA (底部行动号召)**：再次提醒用户采取行动。
- **Footer (底部信息)**：法律声明、社交媒体链接。

### 2. 文案撰写框架 (Copywriting Frameworks)
根据目标群体和产品类型选择合适的文案策略：
- **AIDA (Attention, Interest, Desire, Action)**：引起注意、激发兴趣、创造欲望、促进行动。
- **PAS (Problem, Agitate, Solve)**：指出问题、激化痛苦、给出解决方案。
- **FAB (Features, Advantages, Benefits)**：属性、优势、利益点。
- **Before-After-Bridge (BAB)**：描述现状、展示愿景、产品作为桥梁。

### 3. TSX + Tailwind 代码生成
- **响应式设计 (Responsive)**：代码必须默认适配移动端（Mobile First），使用 `md:`, `lg:` 等前缀进行适配。
- **组件化思维**：将每个 Section 拆分为独立的 React 组件。
- **深色模式支持**：可选支持 `dark:` 模式。
- **交互效果**：建议使用简单的 `framer-motion` 或纯 CSS `transition` 增加页面生动感。

### 4. A/B 测试建议 (Experimental Design)
为同一个落地页提供两个不同的方向：
- **版本 A (理性型)**：强调功能、数据、逻辑。
- **版本 B (情感型)**：强调故事、梦想、共鸣。

### 5. 转化率优化 (CRO) 要素
- **F-Pattern & Z-Pattern 布局**：符合人类视觉扫描习惯。
- **对比色 CTA**：确保按钮颜色在页面中极具辨识度。
- **加载速度优化**：建议使用 Next/Image 优化图片，避免大型第三方脚本阻塞渲染。

## 边界与限制
1. **真实性原则**：严禁生成虚假的社交证明（Fake Testimonials）或欺骗性的倒计时。
2. **后端逻辑缺失**：本技能仅负责 UI 和文案生成，不包含数据库集成、邮件发送等后端逻辑。
3. **法律风险**：生成的文案不代表法律建议，用户需自行审核是否符合当地广告法。
4. **过重依赖**：虽然 AI 可以生成基础框架，但极具品牌特色的视觉设计（Graphic Design）仍需专业设计师介入。

## 最佳实践准则
- **一个页面，一个目标**：不要在一个落地页里推销两个不相关的产品。
- **减少摩擦**：表单字段越少越好，CTA 应该清晰明了。
- **测试一切**：小到一个按钮的文字，大到整体布局，都应基于数据进行优化。
